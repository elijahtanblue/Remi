# Remi Product Direction

Derived quick reference. [Remi-ticket-reconstruction-assistant-v3.md](Remi-ticket-reconstruction-assistant-v3.md) is the singular source of truth for product strategy, scope, ICP, roadmap priorities, and positioning. If this file conflicts with V3, V3 wins.

Remi is the coordination layer for messy issues. When the ticket is no longer the whole story, Remi reconstructs what is actually happening across Jira, Slack, and email, maintains the Current Work Record, and helps the next action happen through controlled updates.

## Current Wedge

- Support, implementation, and customer operations teams handling escalations and vendor/customer issues
- Cross-functional delivery workflows where ownership, blockers, and external context are fragmented
- Guided design-partner pilots with narrow workflow scope

## Product Home

- Web work queue and issue detail page are the workflow home
- Slack is an action surface
- Jira is the issue anchor
- Gmail is part of the core evidence triad

## Build Filter

Prioritize features that strengthen this loop:

1. Select one real issue with fragmented context.
2. Gather scoped Slack, Jira, and Gmail evidence.
3. Build a cited Current Work Record.
4. Show owner, blocker, waiting-on, and next step.
5. Propose a reviewable action or writeback.

Defer broad memory, broad search, Slack-only UX, docs-platform scope, and unrestricted automation.
